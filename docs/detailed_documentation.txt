# CONTRIBUTING.md
# Contributing to MAVERIC

We welcome contributions to MAVERIC! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/yourusername/maveric.git
   cd maveric
   ```
3. Create a virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```
4. Install development dependencies:
   ```bash
   pip install -e ".[dev]"
   ```

## Development Workflow

1. Create a new branch for your feature:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. Make your changes and ensure tests pass:
   ```bash
   make test
   ```

3. Format and lint your code:
   ```bash
   make format
   make lint
   ```

4. Commit your changes:
   ```bash
   git add .
   git commit -m "Add your descriptive commit message"
   ```

5. Push to your fork and create a pull request

## Code Style

- We use [Black](https://github.com/psf/black) for code formatting
- We use [isort](https://pycqa.github.io/isort/) for import sorting
- Follow PEP 8 guidelines
- Add type hints where possible
- Write descriptive docstrings for all public functions and classes

## Testing

- Write tests for new functionality
- Ensure all tests pass before submitting PR
- Aim for high test coverage
- Use pytest fixtures for common test data

## Adding New Features

### Adding a New Quality Metric

1. Create a new metric class in `maveric/quality/metrics/`:
   ```python
   from maveric.quality.metrics.base_metric import BaseQualityMetric
   
   class MyNewMetric(BaseQualityMetric):
       @property
       def metric_name(self) -> str:
           return "my_new_metric"
       
       def compute(self, image: Image.Image, metadata: Dict[str, Any]) -> float:
           # Your implementation
           pass
   ```

2. Add tests in `tests/test_quality_metrics.py`

3. Update documentation

### Adding a New Dataset

1. Create a dataset handler in `maveric/datasets/`:
   ```python
   from maveric.core.base import BaseDataset
   
   class MyDataset(BaseDataset):
       # Implementation
   ```

2. Register it in `maveric/datasets/__init__.py`

3. Add tests and documentation

## Documentation

- Update docstrings for API changes
- Update README.md if needed
- Add examples for new features
- Update API reference documentation

## Pull Request Process

1. Update the README.md with details of changes if applicable
2. Update the documentation with any new APIs
3. The PR must pass all CI checks
4. Request review from maintainers
5. Once approved, it will be merged

## Questions?

Feel free to open an issue for any questions about contributing!

# Installation Guide

## Requirements

- Python 3.8 or higher
- CUDA-capable GPU (recommended)
- At least 8GB RAM

### Using pip

```bash
pip install maveric
```

### From source

```bash
git clone https://github.com/yourusername/maveric.git
cd maveric
pip install -e .
```

### Development installation

```bash
git clone https://github.com/yourusername/maveric.git
cd maveric
pip install -e ".[dev]"
```

## Docker Installation

### Build image

```bash
docker build -t maveric:latest .
```

### Run with GPU support

```bash
docker run --gpus all -it \
    -v $(pwd)/data:/data \
    -v $(pwd)/configs:/configs \
    maveric:latest
```

### Using docker-compose

```bash
docker-compose up
```

## Platform-Specific Instructions

### Linux

Most straightforward installation:

```bash
# Install system dependencies (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install python3-pip python3-dev

# Install MAVERIC
pip install maveric
```

### macOS

```bash
# Install using Homebrew Python
brew install python@3.9
pip3 install maveric
```

Note: GPU acceleration not available on macOS. Will use CPU.

### Windows

```bash
# Using Anaconda (recommended)
conda create -n maveric python=3.9
conda activate maveric
pip install maveric
```

## Verification

Verify installation:

```python
import maveric
print(maveric.__version__)

# Check CLIP model loading
from maveric import MAVERIC
m = MAVERIC()
print("Installation successful!")
```

## Common Issues

### CUDA not found

If you see CUDA-related errors but have a GPU:

```bash
# Install PyTorch with CUDA support
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118
```

### Memory errors

For large-scale processing:
- Reduce batch size in configuration
- Enable disk caching
- Use chunked processing

### Import errors

Ensure all dependencies are installed:

```bash
pip install -r requirements.txt
```

# docs/architecture.md
# MAVERIC Architecture

## System Overview

MAVERIC follows a modular architecture designed for extensibility and maintainability:

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│                 │     │                  │     │                 │
│    Retrieval    │────▶│ Quality Control  │────▶│  Customization  │
│                 │     │                  │     │                 │
└─────────────────┘     └──────────────────┘     └─────────────────┘
         │                       │                         │
         │                       │                         │
         ▼                       ▼                         ▼
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Cache Manager  │     │  Visualizations  │     │    Training     │
└─────────────────┘     └──────────────────┘     └─────────────────┘
```

## Core Components

### 1. Retrieval System

The retrieval system handles:
- Loading source datasets
- Computing multi-modal embeddings
- Scoring samples against reference data
- Managing batch processing

Key classes:
- `Retriever`: Main retrieval engine
- `CacheManager`: Handles image and result caching
- `DatasetHandler`: Abstract interface for datasets

### 2. Quality Control

Quality control includes:
- Multiple quality metrics (visual, semantic, multimodal)
- Configurable filtering pipelines
- Dataset balancing strategies

Key classes:
- `QualityController`: Orchestrates quality assessment
- `BaseQualityMetric`: Abstract base for metrics
- `QualityFilter`: Abstract base for filters

### 3. Model Customization

Customization features:
- Locked-text tuning approach
- Regularization to prevent forgetting
- Flexible training configurations

Key classes:
- `ModelCustomizer`: Manages customization pipeline
- `CustomizedCLIP`: Modified CLIP model
- `Trainer`: Handles training loop

## Design Patterns

### Factory Pattern

Used for creating datasets and models:

```python
dataset = DatasetFactory.create_dataset("cifar100")
model = ModelFactory.create_model("clip", model_name="ViT-B/32")
```

### Strategy Pattern

Quality metrics implement a common interface:

```python
class BaseQualityMetric(ABC):
    @abstractmethod
    def compute(self, image, metadata) -> float:
        pass
```

### Builder Pattern

Configuration uses builder pattern:

```python
config = MAVERICConfig()
    .set_cache_dir("/path/to/cache")
    .set_device("cuda")
    .build()
```

## Data Flow

1. **Retrieval Phase**
   - Load source dataset
   - Compute embeddings for each sample
   - Score against reference embeddings
   - Save results with rotation

2. **Quality Control Phase**
   - Load retrieval results
   - Apply quality metrics
   - Filter based on thresholds
   - Balance dataset if needed

3. **Customization Phase**
   - Create data loaders from filtered data
   - Fine-tune vision encoder
   - Evaluate on target dataset
   - Save best model

## Extension Points

### Adding New Metrics

```python
class MyMetric(BaseQualityMetric):
    def compute(self, image, metadata):
        # Your implementation
        return score
```

### Adding New Datasets

```python
class MyDataset(BaseDataset):
    @property
    def class_names(self):
        return ["class1", "class2"]
    
    def get_reference_samples(self, n_per_class):
        # Return reference images
        pass
```

### Custom Filters

```python
class MyFilter(QualityFilter):
    def apply(self, data):
        # Filter logic
        return filtered_data
```

## Performance Considerations

1. **Caching**: Aggressive caching of images and embeddings
2. **Batch Processing**: Configurable batch sizes
3. **GPU Utilization**: Automatic device selection
4. **Memory Management**: Rotation-based saving for large datasets

## Security Considerations

- No execution of arbitrary code
- Validated input parsing
- Secure default configurations
- Sandboxed file operations
